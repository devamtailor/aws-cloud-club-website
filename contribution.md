# 📢 Contributing to AWS Cloud Club Website

Thank you for your interest in contributing to this project! 🎉
We follow a **Fork & Pull Request workflow** to maintain code quality and ensure smooth collaboration.

---

# 🚀 Contribution Workflow

## 🔗 1. Fork the Repository

* Go to the repository
* Click on **Fork** (top right corner)

---

## 💻 2. Clone Your Fork

```bash
git clone https://github.com/YOUR-USERNAME/aws-cloud-club-website.git
cd aws-cloud-club-website
```

---

## 🔄 3. Add Upstream Repository

```bash
git remote add upstream https://github.com/devamtailor/aws-cloud-club-website.git
```

---

## 🔃 4. Sync Latest Code

```bash
git fetch upstream
git checkout dev
git merge upstream/dev
```

---

## 🌿 5. Create a Feature Branch

```bash
git checkout -b feature/your-feature-name
```

### Examples:

* feature/navbar
* feature/homepage-ui
* feature/backend-api

---

## 🛠️ 6. Make Changes & Commit

```bash
git add .
git commit -m "Describe your changes clearly"
```

---

## 🚀 7. Push to Your Fork

```bash
git push origin feature/your-feature-name
```

---

## 🔀 8. Create Pull Request

* Go to the original repository
* Click on **Compare & Pull Request**
* Set:

  * Base branch → `dev`
  * Compare branch → your feature branch

---

# 📌 Contribution Rules

## 🔹 Branching Rules

* Always create branch from `dev`
* Never work on `main`
* Do not push directly to `main` or `dev`

---

## 🔹 Pull Request Rules

* PR must target `dev` branch
* Add a clear title and description
* Keep PR small and focused
* Resolve all review comments before merging

---

## 🔹 Code Guidelines

* Write clean and readable code
* Maintain proper folder structure
* Avoid unnecessary files and console logs

---

# 🔄 Keeping Your Fork Updated

```bash
git fetch upstream
git checkout dev
git merge upstream/dev
```

---

# 👥 Workflow Overview

```
Fork → Clone → Create Branch → Commit → Push → Pull Request → Review → Merge
```

---
